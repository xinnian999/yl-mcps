## 1.智能切换分支 (done)

背景：因为日常工作的项目开发，基本都是分支开发分支发布。所以每次迭代都需要在远程先创建新的分支，然后本地同步并切换到这个新分支。
（受公司限制只允许在远程手动创建分支，无法通过脚本创建，所以本次只需要实现同步远程并切换分支的功能）

具体需求：实现根据用户输入的分支名称，切换到这个新分支，切换之前需要先同步远程。

## 2.智能提交评审 (done)

背景：公司代码库规范，仅允许 git push origin HEAD:refs/for/xxx 发起评审的方式来提交代码，

### 输入参数

1. 卡片代号
2. 提交分支(未输入的默认为当前分支)

### 提交过程

1. git add .
2. ai根据卡片代号自动创建commit信息，例子：bdife-3679 下线部分功能... 
3. git commit -m "ai自动生成的commit信息"
4. git push origin HEAD:refs/for/提交分支



## 3.修改评审

### 背景

假如上个commit提交了评审，但这个评审还没有通过，也就是代码还没有真的合入到分支。那么我又修改了代码，再提交肯定不想产生新提交，最好是合并到上次一起评审
