# 待实现工具

在实现工具时，需要参考`tool-handlers.js`中的实现方式。

## 1.切换远程分支

背景：因为日常工作的项目开发，基本都是分支开发分支发布。所以每次迭代都需要在远程先创建新的分支，然后本地同步并切换到这个新分支。
（受公司限制只允许在远程手动创建分支，无法通过脚本创建，所以本次只需要实现同步远程并切换分支的功能）

具体需求：实现根据用户输入的分支序号（因为完整分支名称较长，但格式一样，只有序号不同），切换到这个新分支，切换之前需要先同步远程。

### 需要的额外实现的git工具：

#### 1.1 列出远程分支
- 命令：`git branch -r` 或 `git ls-remote --heads`
- 用途：获取所有远程分支列表，用于根据序号匹配完整分支名称

#### 1.2 创建并切换到跟踪远程分支的本地分支
- 命令：`git checkout -b <local-branch> origin/<remote-branch>`
- 用途：当本地没有对应分支时，创建本地分支并设置跟踪关系

#### 1.3 查看分支跟踪关系（可选）
- 命令：`git branch -vv`
- 用途：确认分支是否已经存在并正确跟踪远程分支

#### 1.4 获取当前分支名（辅助）
- 命令：`git rev-parse --abbrev-ref HEAD`
- 用途：获取当前所在分支名称

#### 1.5 检查工作区状态（辅助）
- 命令：`git status --porcelain`
- 用途：切换分支前检查工作区是否干净

#### 1.6 实现常规的git fetch
- 命令：`git fetch`
- 用途：同步远程仓库信息

#### 1.7 实现常规的git checkout
- 命令：`git checkout <branch>`
- 用途：切换到指定分支

### 实现流程：
1. 执行 `git fetch` 同步远程
2. 执行 `git branch -r` 获取远程分支列表
3. 根据用户输入的序号匹配对应的完整分支名称
4. 检查本地是否已有该分支
5. 如果没有：执行 `git checkout -b <local> origin/<remote>`
6. 如果有：执行 `git checkout <local>`
